# LILA BLACK - Player Journey Visualizer

An interactive web tool for visualizing player behavior across LILA BLACK's maps. Built for the Level Design team to explore where players move, fight, die, and loot — without needing to touch raw data.

**Live:** https://lila-viz-web.vercel.app

---

## What It Does

You get a browser-based map viewer that lets you:

- See all player events (positions, kills, deaths, loot, storm deaths) plotted on the actual minimap
- Switch between all 3 maps — Ambrose Valley, Grand Rift, Lockdown
- Filter by date (Feb 10–14), specific match, human vs bot players, and event type
- Toggle a heatmap overlay to spot high-traffic zones or kill hotspots
- Watch a single match play out using the timeline scrubber
- See live stats (total events, kills, deaths, storm deaths, loot, human %, bot %) update as you filter

Everything runs in the browser — no server, no loading spinner after the first fetch.

## Running Locally

### Prerequisites

- Node.js 18+
- Python 3.9+ with pandas and pyarrow

```bash
pip install pandas pyarrow
```

### Step 1 — Clone the repo

```bash
git clone https://github.com/mn409/lila-viz-web.git
cd lila-viz-web
```

### Step 2 — Add your data

Place the `player_data/` folder (with the 1,243 `.nakama-0` files in `February_XX/` subfolders) in the project root:

```
lila-viz-web/
└── player_data/
    ├── February_10/
    ├── February_11/
    ...
```

Also place the 3 minimap images into `public/minimaps/`:
- `AmbroseValley_Minimap.png`
- `GrandRift_Minimap.png`
- `Lockdown_Minimap.jpg`

### Step 3 — Run the preprocessor

```bash
python preprocess_new.py
```

This reads all `.nakama-0` files, maps world coordinates to pixel coordinates, detects bots, and writes 3 JSON files into `public/data/`. Takes about 30 seconds for the full dataset.

Expected output:
```
Found 1243 files
  Processing 0/1243...
  → public/data/AmbroseValley.json  (61,013 events)
  → public/data/GrandRift.json      (6,853 events)
  → public/data/Lockdown.json       (21,238 events)
Done. Total events: 89,104
```

### Step 4 — Start the app

```bash
npm install
npm run dev
```

Open http://localhost:3000

## Data Pipeline

The raw `.nakama-0` files are Parquet-formatted binaries (PAR1 magic bytes) from the Nakama game server. The preprocessor:

1. Reads each file with `pandas.read_parquet()`
2. Decodes byte-encoded event strings (`b'Position'` → `'Position'`)
3. Converts world coordinates to pixel coordinates using map-specific origin and scale values
4. Detects bots by checking if `user_id` is a valid UUID4 (humans) or not (bots)
5. Extracts the date from the parent folder name (`February_10` → `2026-02-10`)
6. Splits output into one JSON file per map for lazy loading

## Notes

- Map data is lazy-loaded — only the selected map's JSON is fetched, then cached in memory
- Heatmap runs entirely in the browser using a separable Gaussian blur — no backend needed
- Bot detection is based on UUID format validation, as documented in the original data README
